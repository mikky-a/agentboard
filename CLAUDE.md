# Agent Board

Локальная доска агентов: `agentboard.py` (сервер) + `index.html` (весь UI) + `skins/` (темы поверх базовой).

## Темы

У доски три темы: базовая «Терминал» (стили прямо в index.html), `skins/macos.css` и `skins/soviet.css`.
**Любые интерфейсные изменения обязаны быть стилизованы под все три темы** — новый элемент или анимация
добавляется в index.html и получает переопределения в обоих скинах, в духе каждой темы
(терминал — фосфорный глоу, macOS — стекло и системные цвета, СССР — лампы/эмаль/ЭЛТ).
Не забывать `@media (prefers-reduced-motion: reduce)` для новых анимаций.

## Статусы агентов

«Ждёт тебя» и остальные статусы приходят из lifecycle-хуков четырёх CLI, которые
пишут `~/.claude/agent-status/<tmux-сессия>`: Claude Code и Codex — событие
PermissionRequest и др.; Cursor — свой `~/.cursor/hooks.json` + скрипт
`~/.cursor/agent-status.sh` (ответ-разрешение из хука Cursor игнорирует, поэтому
tee-имя разрешается через его allowlist `Shell(tee)`); opencode — плагин
`~/.config/opencode/plugins/agent-status.js` (событие permission.asked).
Хуки ставятся с доски (плашка в топбаре → `/api/hooks_install`, код в agentboard.py);
не изобретать определение статусов по тексту панели — только как fallback-эвристику.

## Имена агентов

Новый агент называет свою карточку сам: к его первому сообщению доска дописывает
маркер `[agentboard:<путь>]`, а инструкция (записать короткое имя задачи через `tee`
в `/tmp/agentboard-<uid>-names/<tmux-сессия>`) живёт в глобальной памяти агента —
`~/.claude/CLAUDE.md` / `~/.codex/AGENTS.md`; секции ставит плашка hooks_install.
Без установленной секции — фолбэк: инструкция целиком в хвосте (`name_tail`).
`get_agents` подбирает файл и ставит как `label`; ручное имя всегда главнее.
Маркер и хвост вырезаются из превью (NAME_RE). Имя — только через файл,
не парсить вывод терминала.

## Поведение превью

Превью карточек собирается из структурных логов, не из пейна: jsonl клода,
rollout codex, store.db курсора (`~/.cursor/chats/<md5(cwd)>/<uuid>/`, блобы
читать как bytes — среди них protobuf), общий sqlite опенкода
(`~/.local/share/opencode/opencode.db`, таблицы session/message/part).
У не-клодов пейн не используется вообще (стартовый шум), у клода он только
фолбэк. Тул-вызовы — строками `⏺ Имя(деталь)`, их красит renderPreview.
`card["rollout"]` — «лог разговора»: путь файла у codex/cursor, id сессии у opencode.

При отправке ответа агенту превью терминала визуально очищается: сообщение юзера + новый вывод,
старый вывод до сообщения не сохраняется. Анимации ответа должны это учитывать.

## Терминал агентов

tmux-сессии создаются 220×50 (иначе рождаются 80×24 и TUI мажет при ресайзе)
и с `mouse on` — без него колесо превращается в стрелки и листает историю ввода.
Клод скроллит колесом сам (запрашивает мышь), codex мышь не умеет — его листает
copy-mode tmux; индикатор позиции приглушён через `mode-style`.
